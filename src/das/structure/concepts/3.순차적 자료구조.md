# 순차적 자료구조 (Sequential Data Structure)

데이터를 메모리 공간에 연속적으로 저장하는 가장 기본적인 자료구조. 크게 구현 방식(메모리 구조)에 따라 '배열'과 '연결 리스트'로 나뉘고, 접근 방식(사용 규칙)에 따라 '스택, 큐, 덱'으로 나뉜다.

## 1. 배열과 리스트 (Array & List)

*   기본 특성
    *   연속성: 데이터가 물리적으로 인접하여 저장됨.
    *   임의 접근 (Random Access): 인덱스(Index)를 통해 모든 원소에 즉시 접근 가능 (상수 시간, $O(1)$).
    *   예시: C언어의 배열(Array), 파이썬의 리스트(List).
*   연산 및 수행 시간
    *   삽입/삭제 (맨 뒤): `append`, `pop()`. 추가적인 데이터 이동이 없어 상수 시간($O(1)$) 소요.
        *   수학적 위치 계산이 가능. `시작 주소 + (인덱스 × 데이터 크기)` 공식으로 100만 번째 데이터도 바로 접근 가능.
    *   삽입/삭제 (중간/맨 앞): `insert`, `remove`.
        *   삭제 혹은 삽입된 위치의 빈 공간을 처리하기 위해 기존 데이터들을 앞뒤로 이동(Shift)시키는 연산이 필요.
        *   데이터의 개수($N$)만큼 이동해야 하므로 데이터에 정비례한 선형 시간($O(N)$) 소요.

## 2. 연결 리스트 (Linked List)

데이터를 메모리의 연속되지 않은 공간에 분산 저장하는 자료구조.

*   구조
    *   노드(Node) 구성: 현재 원소의 값(Data) + 다음 원소의 참조값(Next Pointer).
    *   연결 방식: 각 원소가 다음 원소를 가리키며, 마지막 원소는 `NULL (또는 None)`을 참조하여 끝임을 표시.
*   배열과의 비교
    *   접근(탐색): 인덱스를 통한 임의 접근 불가능. 첫 원소부터 링크를 따라 순차적으로 이동해야 하므로 탐색 효율이 낮음 ($O(N)$).
    *   삽입/삭제: 삽입/삭제할 위치의 앞뒤 원소의 참조값(링크)만 변경하면 되므로 데이터 이동 없이 효율적인 처리가 가능.

## 3. 스택, 큐, 덱 (Stack, Queue, Deque)

배열과 리스트를 기반으로 하되, 데이터의 접근(삽입, 삭제) 지점을 제한한 자료구조.

*   스택 (Stack)
    *   구조: LIFO (Last In First Out, 후입선출).
    *   특징: 진입점과 출구가 동일한 구조. 가장 마지막에 삽입(push)된 데이터가 가장 먼저 삭제(pop)됨.
*   큐 (Queue)
    *   구조: FIFO (First In First Out, 선입선출).
    *   특징: 진입점(Rear)과 출구(Front)가 구분된 구조. 가장 먼저 들어온 데이터가 가장 먼저 나감. (대기열 등)
*   덱 (Deque)
    *   구조: 스택과 큐의 결합 (Double-ended Queue).
    *   특징: 양쪽 끝(Front, Rear) 모두에서 삽입과 삭제가 가능한 양방향 구조.
